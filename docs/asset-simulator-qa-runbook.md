# 자산 시뮬레이터 QA Runbook

`/asset-simulator?tab=safety`의 안정성 체크는 등급 대시보드가 아니라, 목표 월생활비를 절세계좌 인출과 위탁계좌 배당으로 얼마나 충당하는지 확인하는 현금흐름 화면이다. 이 문서는 계산식, stress 정책, resolver 정책, 저장 schema를 바꾸지 않는 UI 검증 절차다.

## 사전 조건

- 로컬 저장 또는 로그인 계정에 자산 시뮬레이터 설정이 있다.
- `/asset-simulator?tab=safety`로 직접 진입한다.
- 기본 시나리오와 하락장 시나리오 결과가 준비된다.
- 포트폴리오 예시는 검증용이며 투자 권유가 아니다.

## 화면 순서

상단 단계는 다음 세 개만 보여야 한다.

1. 목표 설정
2. 계좌 입력
3. 결과 확인

`가정 적용`이라는 단계명은 노출하지 않는다. 입력 영역은 결과 영역보다 먼저 나타나며, 결과 확인 버튼을 누르면 기존 적용 경로를 통해 projection을 갱신한 뒤 결과 영역으로 이동한다.

## 목표 설정

1. 목표 월생활비, 기간, 물가상승률 입력이 한 카드에 보이는지 확인한다.
2. 목표 월생활비를 예: `300만원`, 기간을 `20년`, 물가상승률을 `2.5%`로 바꾼다.
3. 시작 연도 입력은 안정성 체크의 핵심 입력에 노출되지 않는지 확인한다. 내부 startYear 기본 동작은 유지한다.
4. `목표 생활비와 자산은 물가상승률을 반영한 현재가치로 비교합니다.` 안내가 보이는지 확인한다.

## 계좌 입력

### 절세계좌 · 인출 기반

1. 절세계좌 총자산과 연 인출률은 직접 입력할 수 있어야 한다.
2. 총자산은 기존 ISA·연금저축 입력을 합산해 보이며, 값을 바꾸면 기존 입력 구조를 유지한 채 합계가 갱신된다.
3. 예상 CAGR과 예상 월 현금은 읽기 전용 텍스트/배지로 보이며 input처럼 보이지 않아야 한다. 비중 합계가 100%가 아니면 확정 숫자 대신 `비중 100% 입력 후 계산` 또는 `확인 필요`가 보여야 한다.
4. SCHD, QLD 등 티커 input은 충분히 넓고 비중 input은 별도로 작게 배치된다.
5. 각 행의 자동 환산 금액은 `계좌 총자산 × 비중`으로 표시되며 수정 input이 아니어야 한다.
6. 기본 행에는 티커, 비중, 자동 환산, CAGR, 가정 수정, 삭제만 보인다.

### 위탁계좌 · 배당 현금흐름 기반

1. 위탁계좌 총자산은 직접 입력할 수 있어야 한다.
2. 예상 세후 배당률과 예상 월 현금은 읽기 전용 표시여야 한다. 종목을 삭제한 뒤에는 남은 종목 기준으로 다시 계산되며, 이전 값이 남아 있지 않아야 한다.
3. 각 행에 티커, 비중, 자동 환산, 배당률·배당성장률·주가성장률 요약, 가정 수정, 삭제가 보이는지 확인한다.
4. 배당률·성장률 기본값은 input처럼 보이지 않아야 한다.

### 공통 입력 UX

1. 티커 또는 비중을 바꾸면 자동 계산 상태가 `자동 계산 중`, `자동 계산 완료`, `가정 일부 확인 필요` 중 하나로 갱신되는지 확인한다.
2. `자동 계산 / 수동 입력` toggle, `전체 자동 계산` 버튼, 종목별 slider, 기본 고정 checkbox가 없는지 확인한다.
3. 종목별 `+ 종목 추가`와 비중 합계 `100% 완료` 또는 `확인 필요` 상태가 보이는지 확인한다.
4. `비중을 수정하면 나머지 종목 비중이 자동 조정됩니다.` 안내가 보이는지 확인한다. 현재 저장된 비중이 100%가 아니면 계산 결과 확인을 막고 `비중 100% 입력 후 계산` 이유를 표시해야 한다.

## 가정 수정

1. 기본 화면에는 수동 가정 input이 보이지 않아야 한다.
2. 절세계좌 행에서 `가정 수정`을 누르면 예상 CAGR input만 펼쳐지는지 확인한다.
3. 위탁계좌 행에서 `가정 수정`을 누르면 배당률, 배당성장률, 주가성장률 input이 펼쳐지는지 확인한다.
4. `자동값으로 되돌리기`가 기존 manual assumptions를 제거하고 자동 resolver 상태로 돌리는지 확인한다.
5. 데이터 부족·조회 실패 시에도 새 저장 schema를 만들지 않고 가정 수정으로만 보완하는지 확인한다.

## 결과 확인과 월생활비 충당

1. primary CTA가 `결과 확인`이고, 그 옆에 secondary `저장` 버튼이 보이는지 확인한다. 입력을 저장한 뒤 새로고침해 기존 저장/복원 흐름이 유지되는지 확인한다.
2. 결과 첫 카드에 목표 월생활비, 총 월 현금, 절세계좌 월 현금, 위탁계좌 월 현금, 월 여유/부족, 충당률이 보이는지 확인한다.
3. 목표 300만원보다 공급이 큰 fixture에서 114% 같은 100% 초과 충당률이 `100%`에서 잘린 것처럼 보이지 않는지 확인한다.
4. 충당률 bar는 150% 기준을 사용하고 100% 기준선이 표시되는지 확인한다.
5. 최종 실질자산과 grade가 상단 핵심 KPI가 아닌지 확인한다.

## 계좌별 결과와 하락장

1. 계좌별 결과 요약에는 월 현금, 기간 후 실질자산, 실가치보존율, 가정 수익률, 기준이 보이는지 확인한다.
2. 화면의 비율 라벨이 모두 `실가치보존율`로 표시되는지 확인한다.
3. 도움말은 물가상승률을 반영한 현재가치 기준 자산 보존율임을 설명해야 한다.
4. 하락장 비교에는 `월 현금: 기본 n만원 → 하락장 n만원`, `목표 대비: 기본 n만원 여유/부족 → 하락장 n만원 여유/부족`, 계좌별 실가치보존율 변화, 부족 시작 연도가 보이는지 확인한다. 음수 기호와 `부족` 문구가 함께 보이면 안 된다.
5. `하락장 손상폭` 표현은 노출하지 않는다.
6. S/A/B/C/D/F 등급은 접힌 안전성 상세 참고 안에서만 볼 수 있고 결과 Hero의 주인공이 아니어야 한다.

## 차트와 상세

1. 결과 아래에 `월 현금 vs 목표 월생활비` 차트가 첫 보조 차트로 보이는지 확인한다.
2. 목표 입력 시 부족 연도 heat strip이 차트 하단에 유지되는지 확인한다.
3. 계좌별 자산 보존 추이와 안전성 상세 참고는 접힘 상태로 시작하는지 확인한다.
4. 새 차트, 계산 엔진, 최적 가능 인출률 계산이 추가되지 않았는지 확인한다.

## 모바일 및 테마

1. 375px 폭에서 목표 설정 → 절세계좌 → 위탁계좌 → 결과 확인 → 월생활비 충당 결과 → 계좌별 결과 → 하락장 비교 → 차트 순서가 유지되는지 확인한다.
2. 티커 input이 한 줄에서 입력 가능한 폭을 유지하고 body 가로 스크롤이 없는지 확인한다.
3. 종목 자동 환산과 자동 가정 요약은 input처럼 보이지 않는지 확인한다.
4. 다크 모드에서 입력, readonly 배지, 표, 차트 축과 tooltip의 대비가 충분한지 확인한다.
5. basic/safety 탭을 왕복해 입력, 자동 조회 상태, 차트가 유지되는지 확인한다.

## 자동 회귀 명령

다음 명령은 PowerShell에서 UTF-8 환경으로 실행한다.

```powershell
npm.cmd run typecheck
npm.cmd run build
npm.cmd run lint
npm.cmd run check:asset-simulator-portfolio-safety-ui
npm.cmd run check:asset-simulator-portfolio-resilience
npm.cmd run check:asset-simulator-stress-scenario
npm.cmd run check:asset-simulator-target-expense
npm.cmd run check:asset-simulator-safety
npm.cmd run check:asset-simulator-golden
npm.cmd run check:asset-simulator-persistence
```

추가 확인이 필요하면 다음도 실행한다.

```powershell
npm.cmd run check:asset-simulator-portfolio-projection
npm.cmd run check:asset-simulator-portfolio-assumptions
npm.cmd run check:asset-simulator-portfolio-resolver
npm.cmd run check:asset-simulator-portfolio-config
```
