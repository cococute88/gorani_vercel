# DIVIDEND-SCHD-GOAL-AND-WITHDRAWAL-MODE-1

## Streamlit 기준

기존 Streamlit 배당 가계부의 `compute_goal_achievement()`는 목표 종목 수량만 보지 않고, 보유 포트폴리오의 USD 환산 평가금액 전체를 목표 종목 현재가로 나눈 `equivalent_target_quantity`를 계산했다. 따라서 `663.7(325) / 3,300주`는 `SCHD 환산주수 663.7주`, `실제 SCHD 보유주수 325주`, `목표 3,300주`를 의미한다.

## Vercel 차이와 수정

기존 Vercel `/dividends`는 목표 티커 행의 수량 합계만 목표 수량으로 나눠 목표 달성률을 계산했다. 이번 수정으로 목표 달성률은 선택된 배당 요약 범위의 평가금액을 SCHD 현재가(KRW)로 나눈 SCHD 환산주수를 기준으로 계산한다.

```txt
SCHD 환산주수 = SCHD 목표에 투입 가능한 평가금액 / 현재 SCHD 가격(KRW)
목표 달성률 = SCHD 환산주수 / 목표 수량 × 100
```

실제 SCHD 주수는 스냅샷에 실제 수량으로 들어온 SCHD 행만 더한다. 평가금액과 현재가로 역산된 추정 수량은 실보유 수량 표시에서 제외한다. 환산주수는 안전장치로 실제 SCHD 주수보다 작지 않게 보정한다.

현재 SCHD 가격 또는 USD/KRW 환율이 없으면 목표 달성률은 계산 불가로 표시한다.

## 3.5% 인출률 모드

`일괄 3.5% 인출률 적용` 토글을 추가했다. OFF 상태는 기존 TTM 배당 추정 로직을 유지한다. ON 상태는 실제 배당 이력을 섞지 않고 현재 배당 요약 범위의 평가금액에 연 3.5%를 곱해 예상 금액으로 표시한다.

```txt
세전 연간 예상 금액 = 대상 평가금액 × 0.035
세전 월평균 예상 금액 = 세전 연간 예상 금액 / 12
세후 연간 예상 금액 = 대상 평가금액 × 0.035 × 0.846
```

세후 토글이 켜져 있으면 기존 배당 페이지의 배당세 계수와 동일한 `1 - 0.154 = 0.846`을 적용한다. 단, UI에 실제 배당 이력을 반영하지 않는 인출률 가정치임을 표시한다.

## 남은 한계

- SCHD 환산 대상 범위는 현재 `/dividends` 요약 카드의 평가금액 범위(위탁만/절세합산 토글)를 따른다.
- 인출률 모드는 배당 과세의 실제 세무 처리를 모델링하지 않고, 기존 화면의 세전/세후 사용감을 맞추기 위한 단순 계수를 적용한다.
- 목표 설정은 이번 범위에서 클라우드 저장하지 않는다.
