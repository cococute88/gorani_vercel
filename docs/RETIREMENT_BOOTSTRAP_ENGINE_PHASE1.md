# 장기 지속 가능성 블록 부트스트랩 엔진 1단계

## 범위

이 단계는 production UI에 성공률을 노출하지 않는다. 기존 Good·Normal·Bad 계산과 `계좌별 안전성 참고` 영역도 변경하지 않는다. 목적은 사용자 장기 가정을 중심으로 유지하면서 역사적 시장 순서만 재표본화하는 순수 계산 엔진, 데이터 계약, 테스트 및 성능 측정 경로를 만드는 것이다.

## 재중심화 수학

단순 퍼센트포인트 덧셈은 큰 음수 편차에서 `-100%` 아래로 내려갈 수 있고 복리 중심도 사용자 CAGR과 일치하지 않는다. 따라서 연간 gross return의 로그를 사용한다.

1. 역사 수익률 `r_t`를 `h_t = log(1 + r_t)`로 변환한다.
2. 역사 편차 `d_t = h_t - mean(h)`를 계산한다.
3. 사용자 CAGR `g`를 `c = log(1 + g)`로 변환한다.
4. `c + d_t`를 기본 이동값으로 사용한다.
5. 연 수익률이 `-99%~300%` 범위를 벗어나지 않도록 log gross를 제한한다.
6. 제한으로 평균이 이동할 수 있으므로 이분법으로 shift를 다시 찾아 전체 관측치의 평균 log gross가 `c`가 되게 한다.

따라서 전체 역사 관측 풀의 기하평균은 사용자 입력 CAGR을 유지한다. 특정 유한 표본의 평균은 블록 추출 결과에 따라 달라질 수 있다. 상한은 데이터 오류나 과도한 사용자 값이 비정상적 복리 폭발을 만들지 않게 하는 엔진 안전장치이며, production 데이터 연결 시 관측 범위 검토가 필요하다.

엔진 결과는 재중심화 series별 목표·결과 기하평균, 상·하한 clipping 개수, clipping 전후 로그 표준편차를 `recenteringDiagnostics`로 반환한다. 중심값이 유지되더라도 clipping 비중이 높아 꼬리 분포가 압축된 데이터는 production 연결 검토에서 식별할 수 있다.

## 배당·분배 현금흐름 정책

검토한 1단계 대안은 다음과 같다.

1. 가격 하락률만큼 배당을 삭감: 가격과 현금흐름을 잘못 동일시하므로 제외한다.
2. 모든 자산에 임의 위기 haircut 적용: 근거 없는 정교화이므로 제외한다.
3. 완결된 배당성장 패턴이 있을 때만 사용자 배당성장률에 log 편차를 적용하고, 없으면 사용자 성장률만 적용: 기본 정책으로 채택한다.

사용자 배당률은 첫 시뮬레이션 연도 배당 기준액을 만든다. 종목별 `최초 투자액 × 사용자 배당률`을 가격 평가잔고와 분리해 추적하고, 이후 배당 기준액에는 배당성장 패턴만 적용한다. 배당은 재투자하지 않으며 15% 세후 금액만 생활비 공급으로 사용한다. 따라서 가격 하락 자체가 다음 해 배당 현금흐름을 같은 비율로 줄이지 않는다.

JEPQ형 인컴전략도 production 기본 지급 배수는 중립값 `1.0`이다. 실증 근거 없는 위기 임계값이나 삭감률은 기본 계산에 포함하지 않는다. 향후 검증된 분배 스트레스 정책은 안정적인 `policyId`와 지급 배수 함수를 가진 `DistributionStressPolicy`로 명시적으로 주입할 수 있다. 주입 정책이 없으면 가격 패턴과 무관하게 배수 `1.0`을 사용한다.

주입 정책은 같은 입력에서 같은 배수를 반환하는 순수·결정론 함수여야 한다. 결과에는 `distributionStressPolicyId`를 포함해 후속 캐시 키가 서로 다른 정책 결과를 혼용하지 않게 한다.

기존 결정론 위탁계좌 계산은 매년 연초 평가잔액에 성장한 배당률을 곱한다. 신규 확률 엔진은 가격 충격과 배당 현금흐름을 독립적으로 이동해야 하므로, 최초 투자액으로 만든 배당 기준액을 배당성장 패턴으로만 갱신한다. 이 차이는 의도된 신규 엔진 계약이다. 두 방식 모두 배당을 평가잔액에서 차감하거나 재투자하지 않고, 세후 배당을 생활비 공급에 한 번만 포함한다.

## 실질금액 기준

- 시작 시점 전체 투자자산은 할인하지 않는다.
- 1년 차 연말부터 실제로 추출된 해당 연도의 인플레이션을 누적한다.
- `실질금액 = 명목금액 / 누적 인플레이션 지수`다.
- 역사 인플레이션도 사용자 기대 인플레이션을 기하 중심으로 재중심화한다.
- 모든 인플레이션 관측치와 사용자 중심값이 0%인 fixture에서는 누적 지수가 정확히 1이고 명목·실질금액이 동일하다.

## 현금흐름과 성공 판정

엔진은 기존 안전성 점수와 `simulate_total_withdraw` proxy를 사용하지 않는다.

- 시작 시점은 은퇴 행이며 첫 시뮬레이션 연도는 `startYear + 1`이다.
- ISA·연금에는 같은 절세계좌 포트폴리오 총수익 패턴을 적용한다.
- 사용자 종목 비중으로 매년 자산군 수익률을 가중 합산하는 현행 포트폴리오 집계 계약을 따른다. 이는 별도 매매 현금흐름을 만들지는 않지만 수익률 계산상 비중이 유지되는 모델이다.
- 2050년까지는 기존 계약처럼 초기 원금 풀을 인출증가율과 경로 인플레이션에 맞춰 배분하고 비과세로 인출한다.
- 2051년 이후 첫 인출은 당시 평가잔액 × 사용자 인출률이며, 이후 사용자 인출증가율로 증가한다.
- 2051년 이후 세율은 ISA 9.9%, 연금 5.5%다.
- 위탁계좌는 가격 성장과 세후 배당 현금흐름을 제공하지만 원금 매도는 하지 않는다.
- 매년 `세후 ISA 인출 + 세후 연금 인출 + 세후 위탁 배당`이 물가 조정된 필수 인출액을 충족해야 한다.
- 잔액이 양수여도 필수 인출액 일부만 지급하면 그 경로는 해당 시점부터 실패다.
- 총 잔액이 0 이하이면 고갈 실패다.

각 경로는 최초 고갈 연도, 최초 인출 부족 연도, 실질 원금 50%·25% 이하 도달 이력과 기간 종료 실질자산을 기록한다. 임계값 도달 뒤 회복해도 이력은 유지한다.

## 5년 블록과 기간 집계

데이터 관측 행 하나에 같은 연도의 인플레이션, 자산군 총수익률, 가격수익률, 선택적 배당성장률을 함께 둔다. 데이터는 누락 없는 연속 연도여야 한다.

블록 시작점은 `0..N-5`에서 시드 PRNG로 선택한다. 한 블록의 5개 행은 원래 순서대로 복사한다. 70년은 블록 14개로 생성하며, 같은 경로의 30·40·50·60·70년 checkpoint를 집계한다.

## production 데이터 계약

`MarketPatternDatasetV1`은 다음 메타데이터를 요구한다.

- schema version
- dataset ID와 version
- `production` 또는 `test_fixture` 용도
- 시작·종료 연도
- 출처 URL, 라이선스, 조회 시각
- 연도별 인플레이션
- 자산군별 총수익률과 가격수익률
- 선택적이고 전체 기간에 완결된 자산군 배당성장률

후속 데이터 공급자는 `MarketPatternDataAdapter.loadDataset()` 비동기 계약으로 이 스키마를 반환한다. 1단계에는 production adapter 구현체를 등록하지 않으며, dataset이 `null`이면 엔진은 미연결 오류를 반환한다.

테스트 fixture는 `allowTestFixture: true`가 없으면 엔진이 거부한다. 현재 저장소에는 production 장기 자산군 데이터가 연결되지 않았으므로 production 성공률은 산출할 수 없다.

후속 데이터 작업은 다음이 필요하다.

1. BLS CPI 등 승인된 인플레이션 원천 adapter
2. 미국 대형주·대형 성장주·배당 가치주의 장기 연간 total/price return 원천과 재배포 권한
3. 데이터 갱신·버전·무결성 정책
4. 배당성장 자료의 완결성 검증
5. 신규 티커의 승인 매핑 및 정책 검토

production 데이터 연결 결과와 검증은 `RETIREMENT_BOOTSTRAP_PRODUCTION_DATA.md`에 기록한다. 이 문서의 1단계 설계 원칙과 UI 비노출 범위는 그대로 유지한다.

## 실행

```powershell
npm.cmd run check:retirement-bootstrap-engine
npm.cmd run benchmark:retirement-bootstrap-engine
```

벤치마크 fixture는 계산 성능 확인용 synthetic 데이터이며 출력 확률은 금융 결과로 해석하면 안 된다.
